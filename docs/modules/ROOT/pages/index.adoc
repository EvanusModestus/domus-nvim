= Domus Instrumentum
:description: A hierarchical, modular Neovim configuration with 90+ plugins, layered architecture, and embedded documentation
:navtitle: Overview
:icons: font

A modern, hierarchical Neovim configuration built on {neovim-version} with 90+ curated plugins, a 5-layer architecture, and full Antora documentation.

== Philosophy

* **Hierarchical Structure** - Organized by domain: config, core, plugins, features, UI
* **Separation of Concerns** - Plugin specs (what to load) live apart from configs (how to configure)
* **Native Neovim** - Uses `ftplugin/`, `after/`, and Neovim {neovim-version} APIs
* **Documentation First** - Embedded Antora site you are reading right now
* **Lazy Loading** - Fast startup via {plugin-manager} with event-driven loading

== Quick Start

[source,bash,subs=attributes+]
----
# Backup existing config
mv {config-dir} {config-backup}

# Clone
git clone {repo-url} {config-dir}

# Launch (plugins auto-install on first start)
nvim
----

TIP: For side-by-side installs, see xref:installation.adoc[Installation] for the `NVIM_APPNAME` method.

== Structure

[source,subs=attributes+]
----
{repo-name}/
├── init.lua              # Entry point (version check + bootstrap)
├── {lua-dir}/            # Configuration namespace
│   ├── config/           # Layer 1: Options, keymaps, autocmds
│   ├── core/             # Layer 2: lazy.nvim bootstrap, utilities
│   ├── plugins/          # Layer 3: Plugin system
│   │   ├── specs/        # Declarations (what to load)
│   │   │   └── lang/     # Language-specific plugins
│   │   └── config/       # Configurations (how to configure)
│   │       └── lang/     # Language-specific configs
│   ├── features/         # Layer 4: Custom systems (MOTD)
│   └── ui/               # Layer 5: Visual enhancements
├── {ftplugin-dir}/       # Native filetype settings
├── after/syntax/         # Syntax overrides
└── {docs-dir}/           # This Antora documentation
----

See xref:architecture.adoc[Architecture] for the full layer breakdown.
