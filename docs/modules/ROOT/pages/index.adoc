= Domus Neovim
:description: A modular, layered Neovim configuration with 90+ plugins, 5-layer architecture, and embedded Antora documentation
:navtitle: Overview
:icons: font

A modular, layered Neovim configuration built on {neovim-version} with 90+ curated plugins and full Antora documentation.
Designed for polyglot development — Lua, Python, Rust, C/C++, TypeScript, shell, infrastructure-as-code, and technical writing in AsciiDoc and Markdown.

== Design Principles

Separation of concerns::
Plugin _specs_ (what to load) live apart from plugin _configs_ (how to configure).
Adding a plugin never touches configuration logic and vice versa.

Layered boot sequence::
Five layers load in strict order: config -> core -> plugins -> features -> UI.
Each layer has a single responsibility and a clean boundary.

Native Neovim first::
Filetype settings use `ftplugin/`, syntax overrides use `after/`, and LSP configuration uses the Neovim {neovim-version} `vim.lsp.config` API.
Plugins extend — they don't replace — built-in mechanisms.

Lazy by default::
Every plugin loads on demand via {plugin-manager} — by event, filetype, command, or keymap.
Startup is fast regardless of the total plugin count.

Documentation as code::
This Antora site is embedded in the repository.
Documentation stays in sync with the configuration it describes.

== Quick Start

[source,bash,subs=attributes+]
----
# Backup existing config
mv {config-dir} {config-backup}

# Clone
git clone {repo-url} {config-dir}

# Launch with NVIM_APPNAME
NVIM_APPNAME={appname} nvim
----

On first launch, {plugin-manager} bootstraps itself, downloads ~90 plugins, {lsp-installer} installs LSP servers and formatters, and {treesitter} compiles parsers.
Let the initial install complete before editing files.

TIP: For side-by-side installs or platform-specific setup, see xref:installation.adoc[Installation].

== Structure

[source,subs=attributes+]
----
{repo-name}/
├── init.lua              # Entry point (version check + bootstrap)
├── {lua-dir}/            # Configuration namespace
│   ├── config/           # Layer 1: Options, keymaps, autocmds
│   ├── core/             # Layer 2: lazy.nvim bootstrap, utilities
│   ├── plugins/          # Layer 3: Plugin system
│   │   ├── specs/        # Declarations (what to load)
│   │   │   └── lang/     # Language-specific plugins
│   │   └── config/       # Configurations (how to configure)
│   │       ├── lsp/      # LSP server setup
│   │       ├── completion/
│   │       ├── dap/      # Debug adapter setup
│   │       └── lang/     # Language-specific configs
│   ├── features/         # Layer 4: Custom systems (MOTD)
│   └── ui/               # Layer 5: Visual enhancements
├── {ftplugin-dir}/       # Native filetype settings
├── after/syntax/         # Syntax overrides
└── {docs-dir}/           # This Antora documentation
----

See xref:architecture.adoc[Architecture] for the full layer breakdown and design rationale.

== Requirements

[cols="1,1,2", options="header"]
|===
| Component | Version | Notes

| Neovim
| {neovim-version}
| Required — `vim.lsp.config` API

| Git
| 2.19+
| Plugin installation via {plugin-manager}

| C compiler
| gcc or clang
| Required — telescope-fzf-native and LuaSnip jsregexp build natively

| Nerd Font
| Any https://www.nerdfonts.com[Nerd Font]
| Required — icons in statusline, file browser, dashboard, which-key

| Node.js
| 18+
| Optional — for LSP servers (pyright, ts_ls, jsonls, yamlls, etc.)

| ripgrep
| Any
| Recommended — Telescope live grep

| fd
| Any
| Recommended — faster file finding
|===
