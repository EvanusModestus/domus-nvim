:navtitle: Treesitter
:icons: font

== Overview

{treesitter} provides syntax-aware highlighting, code navigation, and structural editing through incremental parsing.
domus-nvim extends the base treesitter installation with several companion plugins:

* **nvim-treesitter-textobjects** -- structural text objects and motions based on the parse tree
* **nvim-treesitter-context** -- sticky scope header pinned to the top of the window
* **rainbow-delimiters.nvim** -- colored bracket pairs by nesting depth
* **hlargs.nvim** -- distinct highlighting for function arguments
* **nvim-autopairs** -- treesitter-aware auto-pairing of brackets and quotes
* **nvim-ts-autotag** -- auto-close and auto-rename HTML/JSX tags

== Ensured Parsers

The following 24 parsers are installed automatically:

[.compact]
`bash`, `c`, `css`, `csv`, `dockerfile`, `html`, `htmldjango`, `http`, `javascript`, `json`, `lua`, `markdown`, `markdown_inline`, `python`, `query`, `regex`, `rust`, `sql`, `toml`, `typescript`, `vim`, `vimdoc`, `xml`, `yaml`

Additional parsers can be installed on demand with `:TSInstall <language>`.

== Text Objects

Provided by **nvim-treesitter-textobjects**, these operate on structural nodes in the parse tree rather than simple text patterns.

== Treesitter Context

**nvim-treesitter-context** displays a sticky header at the top of the window showing the current function, class, or block scope.
When you scroll deep into a function body, the function signature remains visible, providing constant awareness of your position in nested code.

== Rainbow Delimiters

**rainbow-delimiters.nvim** colors matching bracket pairs (`()`, `[]`, `{}`) by nesting depth.
Each depth level receives a distinct color, making it straightforward to identify which opening bracket matches which closing bracket in deeply nested expressions.
