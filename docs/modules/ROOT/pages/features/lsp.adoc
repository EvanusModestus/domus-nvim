= LSP Configuration

:navtitle: LSP
:icons: font

== Overview

domus-nvim provides a complete LSP stack built on three layers:

* **{lsp-installer}** -- automatic server installation and lifecycle management
* **mason-lspconfig.nvim** -- bridge that maps {lsp-installer} package names to lspconfig server names
* **nvim-lspconfig** -- canonical LSP client configuration
* **glance.nvim** -- peek UI for definitions, references, type definitions, and implementations

Completion capabilities are injected from {completion-engine}, ensuring every server advertises snippet support, resolve support, and other extended client capabilities.

On Termux (Android), where {lsp-installer} is unavailable, domus-nvim falls back to the native `vim.lsp.config` API introduced in Neovim 0.11, configuring servers directly without mason-lspconfig.

== Per-Server Customizations

=== lua_ls

Disables the `Lua.diagnostics.libraryFiles` third-party check.
This prevents false positives when editing Neovim plugin code that references runtime libraries.

=== pyright

Type checking mode is set to `basic` rather than `strict`.
Several diagnostic severities are overridden (e.g., `reportMissingImports`, `reportUnusedVariable`) to avoid duplicate diagnostics when ruff is also active as a linter.

=== rust_analyzer

* Check command set to `clippy` for lint-level feedback on save
* All Cargo features enabled (`features = "all"`)
* Inlay hints enabled for types, parameters, and chaining

=== jsonls / yamlls

Both servers integrate with SchemaStore, automatically applying JSON Schema or YAML Schema validation based on the file name and content patterns.

=== on_attach Mappings

|===
| Key | Action

| `gd`
| Go to definition

| `gD`
| Go to declaration

| `gi`
| Go to implementation

| `go`
| Go to type definition

| `gr`
| List references

| `K`
| Hover documentation

| `<C-k>` (insert mode)
| Signature help

| `{leader}vd`
| Show line diagnostics in a float

| `[d` / `]d`
| Previous / next diagnostic

| `{leader}ca`
| Code action

| `{leader}rn`
| Rename symbol

| `{leader}ih`

== Adding a New Server

To add a new language server:

. Add the server name to the `ensure_installed` list in `{lua-dir}/plugins/config/lsp/init.lua`.
. If the server needs custom settings, add a named handler function:
+
[source,lua]
----
-- In the handlers table:
["new_server"] = function()
    require("lspconfig").new_server.setup({
        on_attach = on_attach,
        capabilities = capabilities,
        settings = {
            -- server-specific settings
        },
    })
end,
----
+
. Run `:Mason` to verify the server installs, then open a file of the target filetype to confirm attachment.
