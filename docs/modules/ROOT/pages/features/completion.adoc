= Completion
:description: Auto-completion in domus-nvim powered by blink.cmp, a Rust-based completion engine with LuaSnip snippets and multi-source support.
:navtitle: Completion
:icons: font

== Overview

domus-nvim uses **{completion-engine}** as its completion engine.
Unlike nvim-cmp (written in Lua), {completion-engine} has its core implemented in Rust, resulting in measurably faster fuzzy matching and reduced frame drops during completion in large files.

== Sources

[cols="1,2,1", options="header"]
|===
| Source | Description | Priority

| `lsp`
| Completions from attached language servers
| Default

| `path`
| File system path completions
| Default

| `snippets`
| {snippet-engine} snippet triggers
| Default

| `buffer`
| Words from the current buffer
| Default

| `lazydev`
| Neovim Lua API completions (vim.*, vim.api.*, etc.)
| +100 (boosted above all other sources)
|===

The lazydev source receives a priority boost of +100, ensuring Neovim API completions always appear at the top when editing Lua files within a Neovim configuration.

| Hide completion menu

| Accept selected completion

| `<Tab>`
| Select next item

| `<S-Tab>`
| Select previous item

| `<C-n>`
| Select next item

| `<C-p>`
| Select previous item

| `<C-b>`
| Scroll documentation window backward

| `<C-f>`
| Scroll documentation window forward
|===

== Features

Auto-brackets:: {completion-engine} automatically inserts matching brackets after function completions, saving a keystroke on every function call.

Documentation auto-show:: The documentation window appears automatically after a 200ms delay when a completion item is highlighted, without requiring a manual trigger.

Signature help:: Function signature information displays inline as you type arguments, showing parameter types and positions.

== Snippets

{snippet-engine} serves as the snippet expansion backend, with **friendly-snippets** providing a large pre-made collection covering most languages and frameworks.

Snippet navigation uses `<Tab>` to jump forward through placeholders and `<S-Tab>` to jump backward.
