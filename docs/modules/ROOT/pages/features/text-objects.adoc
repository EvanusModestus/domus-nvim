= Text Objects
:description: Custom text objects for AsciiDoc blocks, Markdown fenced code, treesitter-based selections, and buffer-wide operations via mini.ai and nvim-treesitter-textobjects.
:navtitle: Text Objects
:icons: font

{text-objects} extends Neovim's built-in text objects with custom patterns for AsciiDoc, Markdown, and treesitter-based structural selections.
All objects work with every standard operator: `y` (yank), `d` (delete), `c` (change), `v` (visual select).

== AsciiDoc Text Objects

Custom patterns for AsciiDoc block delimiters and inline formatting:

=== Block Delimiters

[cols="1,2,3", options="header"]
|===
| Object | Delimiter | Example

| `i-` / `a-`
| `----` (listing/code block)
| `yi-` yanks the code inside a listing block

| `i/` / `a/`
| `////` (comment block)
| `da/` deletes an entire comment block with delimiters

| `i=` / `a=`
| `====` (example/sidebar block)
| `ci=` changes the content of an example block
|===

=== Inline Formatting

[cols="1,2,3", options="header"]
|===
| Object | Delimiter | Example

| `i_`
| `+__+` (italic)
| `yi_` yanks italic text without delimiters

| `i*`
| `+**+` (bold)
| `di*` deletes bold text without delimiters

| `i+`
| `++` (passthrough)
| `ci+` changes passthrough content
|===

== Markdown Text Objects

[cols="1,2,3", options="header"]
|===
| Object | Delimiter | Example

| `i{backtick}` / `a{backtick}`
| `{backtick}{backtick}{backtick}` (fenced code block)
| `yi{backtick}` yanks the content of a fenced code block
|===

== Treesitter Text Objects

Provided by nvim-treesitter-textobjects.
These operate on language-aware structural nodes parsed by treesitter:

[cols="1,2", options="header"]
|===
| Object | Target

| `if` / `af`
| Inner / around function

| `ic` / `ac`
| Inner / around class

| `ia` / `aa`
| Inner / around parameter

| `ii` / `ai`
| Inner / around conditional

| `il` / `al`
| Inner / around loop

| `ib` / `ab`
| Inner / around block

| `i/` / `a/`
| Inner / around comment
|===

See xref:features/treesitter.adoc[Treesitter] for full details.

== Special Objects

[cols="1,2", options="header"]
|===
| Object | Target

| `ig`
| Entire buffer (all content)
|===

== Movement Keymaps

Jump between treesitter nodes with bracket motions:

[cols="1,3", options="header"]
|===
| Key | Action

| `]f` / `[f`
| Next / previous function start

| `]c` / `[c`
| Next / previous class start

| `]b` / `[b`
| Next / previous block start
|===

=== Parameter Swapping

[cols="1,3", options="header"]
|===
| Key | Action

| `sn`
| Swap parameter forward (move right in parameter list)

| `sp`
| Swap parameter backward (move left in parameter list)
|===

== Usage Examples

Practical combinations showing how text objects compose with operators:

[cols="2,3", options="header"]
|===
| Command | Effect

| `yi-`
| Yank the contents of an AsciiDoc code block (without `----` delimiters)

| `da=`
| Delete an entire example block including its `====` delimiters

| `ci{backtick}`
| Change the contents of a Markdown fenced code block

| `vaf`
| Visually select an entire function (including signature)

| `daa`
| Delete a function parameter and its trailing comma

| `yig`
| Yank the entire buffer
|===

== Configuration

Text object definitions in `{lua-dir}/plugins/config/mini-ai.lua`.
Treesitter text objects declared in `{lua-dir}/plugins/specs/editor.lua`.
