= Formatting & Linting
:description: Code formatting with conform.nvim and linting with nvim-lint in domus-nvim, covering 10+ languages with format-on-save and auto-lint.
:navtitle: Formatting
:icons: font

== Overview

domus-nvim separates formatting and linting into two dedicated plugins:

* **conform.nvim** -- formatting with per-filetype formatter configuration
* **nvim-lint** -- asynchronous linting with per-filetype linter configuration

All formatters and linters are auto-installed through {lsp-installer}, so no manual binary management is needed.

=== Format on Save

Format-on-save is enabled by default with two exceptions:

* **SQL** -- excluded because sqlfluff can be slow on large queries
* **Markdown** -- excluded to preserve intentional formatting choices

Additionally, format-on-save is skipped when the buffer contains error-level diagnostics, preventing the formatter from mangling code that does not parse correctly.

=== Manual Format

`{leader}cf` -- Format the current buffer on demand, regardless of filetype exclusions.

=== Auto-Lint Triggers

Linting runs automatically on two events:

* **BufWritePost** -- after saving a file
* **InsertLeave** -- after exiting insert mode

This provides near-real-time lint feedback without blocking the editor during typing.

== Mason Auto-Installation

All formatters and linters listed above are managed by {lsp-installer}.
On first launch, {lsp-installer} downloads and installs every configured tool.
Run `:Mason` to view installation status and manage tools manually.

== Configuration

[cols="1,2", options="header"]
|===
| File | Purpose

| `{lua-dir}/plugins/config/conform.lua`
| Formatter-to-filetype mapping, format-on-save logic, shfmt flags

| `{lua-dir}/plugins/config/lint.lua`
| Linter-to-filetype mapping, auto-lint autocommands

| `{lua-dir}/plugins/specs/coding.lua`
| Plugin declarations for conform.nvim and nvim-lint
|===
