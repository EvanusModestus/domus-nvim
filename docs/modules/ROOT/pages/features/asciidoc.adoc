= AsciiDoc Support
:description: Extensive AsciiDoc editing environment with 80+ snippets, structural navigation, editing keymaps, custom commands, text objects, and filetype settings.
:navtitle: AsciiDoc
:icons: font

Comprehensive AsciiDoc editing support spanning 1100+ lines of configuration.
Provides snippets for every common construct, structural navigation, editing keymaps, custom commands for document analysis, and text objects for block-level selections.

== Navigation Keymaps

Jump between structural elements in AsciiDoc documents:

[cols="1,3", options="header"]
|===
| Key | Action

| `]-` / `[-`
| Next / previous `----` block (listing/code)

| `]=` / `[=`
| Next / previous `====` block (example/sidebar)

| `]]` / `[[`
| Next / previous section header

| `]i` / `[i`
| Next / previous `include::` directive

| `]x` / `[x`
| Next / previous `xref:` reference

| `]d` / `[d`
| Next / previous diagram block
|===

== Commands

Custom commands for document analysis and navigation:

`:AsciidocOutline`::
Opens a floating window showing the document's section structure.
Navigate to any section with `Enter`.
Includes a preview of each section's first line.

`:AsciidocWordCount`::
Counts words in the document, excluding AsciiDoc syntax (attributes, directives, delimiters).
Displays the count in a notification.

`:AsciidocFindBrokenXrefs`::
Checks all `xref:` targets against files on disk.
Broken references are listed in a floating window with their line numbers.

`:AsciidocCheckAttrs`::
Finds undefined `+{attribute}+` references in the document.
Knows built-in AsciiDoc attributes and skips them -- only reports genuinely missing definitions.

`:AsciidocPreviewAttrs`::
Displays all `:attribute:` definitions found in the current document in a floating window.

`:AsciidocListDiagrams`::
Lists all diagram blocks (d2, mermaid, plantuml, graphviz, kroki, bpmn) with their titles and line numbers.

== Text Objects

Provided by {text-objects} integration.
See xref:features/text-objects.adoc[Text Objects] for the full reference.

[cols="1,3", options="header"]
|===
| Object | Target

| `i-` / `a-`
| Inside / around `----` blocks (listing/code)

| `i/` / `a/`
| Inside / around `////` blocks (comment)

| `i=` / `a=`
| Inside / around `====` blocks (example/sidebar)

| `i_`
| Inside `+__+` (italic)

| `i*`
| Inside `+**+` (bold)

| `i+`
| Inside `++` (passthrough)
|===

== Filetype Settings

Defined in `{ftplugin-dir}/asciidoc.lua`:

[cols="1,3", options="header"]
|===
| Setting | Behavior

| Spell checking
| Enabled with `en_us` dictionary

| Text wrapping
| Soft wrap with `linebreak`

| Conceal
| Level 2, concealment cursor set to empty string

| Folding
| Indent-based folding available (disabled by default)

| `gf` mapping
| Follows `xref:` and `include::` directives to their target files
|===
