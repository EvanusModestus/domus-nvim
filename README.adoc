= Domus Instrumentum
:description: Hierarchical, modular Neovim configuration
:icons: font

A modern, hierarchical Neovim configuration with embedded documentation.

== Features

* **Hierarchical Structure** - Organized by domain (config, core, plugins, features, ui)
* **Separated Concerns** - Plugin specs vs configs
* **Native ftplugin** - Filetype settings via Neovim's native mechanism
* **Embedded Antora Docs** - Documentation as code

== Quick Start

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-instrumentum ~/.config/nvim
nvim
----

Or run alongside existing config:

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-instrumentum ~/.config/nvim-domus
NVIM_APPNAME=nvim-domus nvim

# Create an alias
alias v='NVIM_APPNAME=nvim-domus nvim'
----

== Key Bindings

Leader: `<Space>`

=== Navigation

* `<Space>ff` - Find files
* `<Space>fg` - Live grep
* `<Space>e` - File browser (Oil)
* `<C-e>` - Harpoon menu

=== LSP

* `gd` - Go to definition
* `gr` - References
* `K` - Hover docs
* `<Space>ca` - Code action

=== Git

* `<Space>gs` - Git status
* `<Space>lg` - LazyGit
* `<Space>hs` - Stage hunk

== Documentation

Full documentation in `docs/` directory.

=== Local Preview

----
cd docs && npx antora antora-playbook.yml
# Open build/site/instrumentum/index.html
----
