= domus-nvim
:description: A modular, layered Neovim configuration with 90+ plugins, 5-layer architecture, and embedded Antora documentation
:icons: font
:url-docs: https://docs.architectus.dev
:url-repo: https://github.com/EvanusModestus/domus-nvim

A modular, layered Neovim configuration built on Neovim 0.11+ with 90+ curated plugins and full Antora documentation.
Designed for polyglot development — Lua, Python, Rust, C/C++, TypeScript, PowerShell, shell, SQL, infrastructure-as-code, and technical writing in AsciiDoc and Markdown.

== Architecture

Five layers load in sequence.
Each layer has a single responsibility and a clean boundary.

[mermaid]
----
graph TD
    A["<b>Layer 1: Config</b><br/>options · keymaps · autocmds"] --> B
    B["<b>Layer 2: Core</b><br/>lazy.nvim bootstrap · utilities"] --> C
    C["<b>Layer 3: Plugins</b><br/>specs — what to load<br/>config — how to configure"] --> D
    D["<b>Layer 4: Features</b><br/>custom systems · MOTD"] --> E
    E["<b>Layer 5: UI</b><br/>visual enhancements"]

    style A fill:#a6e3a1,color:#1e1e2e
    style B fill:#89b4fa,color:#1e1e2e
    style C fill:#cba6f7,color:#1e1e2e

    style E fill:#f38ba8,color:#1e1e2e

Plugin specs (what to load) live apart from plugin configs (how to configure).
Adding a plugin never touches configuration logic and vice versa.

[mermaid]
----

    specs --> config

== Installation

=== Quick Start

[source,bash]
----
# Back up existing config
mv ~/.config/nvim ~/.config/nvim.bak
mv ~/.local/share/nvim ~/.local/share/nvim.bak

# Clone
git clone https://github.com/EvanusModestus/domus-nvim.git ~/.config/domus-nvim

# Launch with NVIM_APPNAME
NVIM_APPNAME=domus-nvim nvim
----

On first launch, lazy.nvim bootstraps itself, downloads ~90 plugins, Mason installs LSP servers and formatters, and Treesitter compiles parsers.
Let it finish before editing.

=== Side-by-Side with Existing Config

[source,bash]
----
alias v='NVIM_APPNAME=domus-nvim nvim'
----

=== Platform Packages

**Arch Linux:**

[source,bash]
----
sudo pacman -S neovim git base-devel ripgrep fd lazygit tmux npm python ipython ttf-jetbrains-mono-nerd
----

**Ubuntu/Debian:**

[source,bash]
----
sudo add-apt-repository ppa:neovim-ppa/unstable
sudo apt update && sudo apt install -y neovim git build-essential ripgrep fd-find nodejs npm python3
sudo ln -sf $(which fdfind) /usr/local/bin/fd
----

**macOS:**

[source,bash]
----
brew install neovim git ripgrep fd lazygit tmux node python ipython
brew install --cask font-jetbrains-mono-nerd-font
----

=== PowerShell & Azure Tooling

The `powershell_es` LSP self-enables only when `pwsh` is on `PATH`.
Completion covers only cmdlets present on `PSModulePath` — the `Az` module is a separate, per-machine install (it does not travel with the synced config).

Install PowerShell Core:

[source,bash]
----
# Arch Linux (AUR)
paru -S powershell-bin

# Ubuntu/Debian
sudo apt install -y powershell

# macOS
brew install --cask powershell
----

Install the Azure module for `Get-Az*` completion (drive `pwsh` non-interactively from your shell):

[source,bash]
----
# Verify absence
pwsh -NoProfile -Command 'Get-Module -ListAvailable Az.Accounts | Select-Object Name, Version'

# Install for the current user (~5 min). -NoProfile so install errors surface.
pwsh -NoProfile -Command 'Install-Module Az -Scope CurrentUser -Repository PSGallery -Force -AllowClobber'

# Confirm on PSModulePath, then restart nvim so PSES rebuilds its command cache
pwsh -NoProfile -Command 'Get-Module -ListAvailable Az.Accounts, Az.Compute | Select-Object Name, Version'
----

PowerShell completion is space-driven (parameters, pipeline cmdlets), so blink is configured to allow space as a completion trigger for `ps1`/`psm1`/`psd1` buffers only — typing `Get-Process | ` opens the pipeline-cmdlet menu.

=== Verify

[source,bash]
----
for cmd in nvim git gcc rg fd lazygit node python3; do
  printf "%-10s %s\n" "$cmd" \
    "$(command -v $cmd >/dev/null 2>&1 && echo 'OK' || echo 'MISSING')"
done
----

Inside Neovim: `:Lazy` (plugins loaded?), `:Mason` (servers installed?), `:checkhealth` (any errors?).
