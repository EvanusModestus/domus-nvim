= domus-nvim
:description: Hierarchical, modular Neovim configuration
:icons: font

A modern, hierarchical Neovim configuration with embedded documentation.

== Features

* **Hierarchical Structure** - Organized by domain (config, core, plugins, features, ui)
* **Separated Concerns** - Plugin specs vs configs
* **Native ftplugin** - Filetype settings via Neovim's native mechanism
* **Embedded Antora Docs** - Documentation as code

=== Installing prerequisites

==== Arch Linux

[source,bash]
----
sudo pacman -S neovim git base-devel ripgrep fd lazygit tmux npm python ipython
----

Install a Nerd Font (e.g., JetBrainsMono):

[source,bash]
----
sudo pacman -S ttf-jetbrains-mono-nerd
----

==== Ubuntu/Debian

[source,bash]
----
# Neovim (PPA for 0.11+)
sudo add-apt-repository ppa:neovim-ppa/unstable
sudo apt update
sudo apt install neovim git build-essential ripgrep fd-find nodejs npm python3 python3-pip

# fd is installed as fdfind on Debian/Ubuntu вЂ” alias it
sudo ln -sf $(which fdfind) /usr/local/bin/fd
----

lazygit (not in default repos):

[source,bash]
----
LAZYGIT_VERSION=$(curl -s "https://api.github.com/repos/jesseduffield/lazygit/releases/latest" | grep -Po '"tag_name": "v\K[^"]*')
curl -Lo lazygit.tar.gz "https://github.com/jesseduffield/lazygit/releases/latest/download/lazygit_${LAZYGIT_VERSION}_Linux_x86_64.tar.gz"
tar xf lazygit.tar.gz lazygit
sudo install lazygit /usr/local/bin
----

Nerd Font вЂ” download from https://www.nerdfonts.com/font-downloads[nerdfonts.com] and install to `~/.local/share/fonts/`, then run `fc-cache -fv`.

==== macOS

[source,bash]
----
brew install neovim git ripgrep fd lazygit tmux node python ipython
brew install --cask font-jetbrains-mono-nerd-font
----

==== Language toolchains

Mason auto-installs most LSP servers and formatters, but some languages require their own toolchain installed first.

**Rust** вЂ” rust-analyzer, rustfmt, and clippy all come from rustup:

[source,bash]
----
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
rustup component add rust-analyzer clippy
----

**Python** вЂ” pyright is installed by Mason, but you need the runtime and optionally ipython for the REPL (`<Space>sc` to send code):

[source,bash]
----
pip install ipython
----

**Node.js** вЂ” required by several Mason-installed servers (ts_ls, html, cssls, jsonls, yamlls). Install via your package manager (shown above) or https://github.com/nvm-sh/nvm[nvm].

Build Neovim from source: https://github.com/neovim/neovim/blob/master/BUILD.md

== Installation

=== Back up existing config

If you have an existing Neovim configuration, move it out of the way first:

[source,bash]
----
mv ~/.config/nvim ~/.config/nvim.bak
mv ~/.local/share/nvim ~/.local/share/nvim.bak
mv ~/.local/state/nvim ~/.local/state/nvim.bak
mv ~/.cache/nvim ~/.cache/nvim.bak
----

=== Clone and launch

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-nvim ~/.config/nvim
nvim
----

Or run alongside an existing config using `NVIM_APPNAME`:

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-nvim ~/.config/nvim-domus
NVIM_APPNAME=nvim-domus nvim

# Create an alias for convenience
alias v='NVIM_APPNAME=nvim-domus nvim'
----

=== First launch

On the first run:

1. **lazy.nvim** bootstraps itself automatically (cloned from GitHub)
2. **~90 plugins** are downloaded and installed вЂ” this takes a minute or two
3. **Mason** installs LSP servers, formatters, and linters in the background
4. **TreeSitter** parsers compile on first encounter of each filetype

Let the initial install complete before editing files. You can check Mason progress with `:Mason` and lazy.nvim status with `:Lazy`.

=== Uninstall

Remove the configuration and all associated data:

[source,bash]
----
rm -rf ~/.config/nvim
rm -rf ~/.local/share/nvim
rm -rf ~/.local/state/nvim
rm -rf ~/.cache/nvim
----

If installed with `NVIM_APPNAME=nvim-domus`:

[source,bash]
----
rm -rf ~/.config/nvim-domus
rm -rf ~/.local/share/nvim-domus
rm -rf ~/.local/state/nvim-domus
rm -rf ~/.cache/nvim-domus
----

== Key Bindings

Leader: `<Space>`

=== Navigation

* `<Space>ff` - Find files
* `<Space>fg` - Live grep
* `<Space>e` - File browser (Oil)
* `<C-e>` - Harpoon menu

=== LSP

* `gd` - Go to definition
* `gr` - References
* `K` - Hover docs
* `<Space>ca` - Code action

=== Git

* `<Space>gs` - Git status
* `<Space>lg` - LazyGit
* `<Space>hs` - Stage hunk
