= domus-nvim
:description: Hierarchical, modular Neovim configuration
:icons: font

A modern, hierarchical Neovim configuration with embedded documentation.

== Requirements

[cols="1,2,3"]
|===
| Component | Version | Notes

| **Neovim**
| 0.11.0+
| Required for `vim.lsp.config` API (mason-lspconfig)

| **Git**
| 2.19+
| For plugin installation via lazy.nvim

| **Node.js**
| 18+
| Optional - for LSP servers (pyright, tsserver, etc.)

| **ripgrep**
| Any
| Optional - for Telescope live grep
|===

=== Neovim Installation

Ubuntu/Debian (PPA for latest):

[source,bash]
----
sudo add-apt-repository ppa:neovim-ppa/unstable
sudo apt update && sudo apt install neovim
----

Arch Linux:

[source,bash]
----
sudo pacman -S neovim
----

macOS:

[source,bash]
----
brew install neovim
----

Build from source: https://github.com/neovim/neovim/blob/master/BUILD.md

== Features

* **Hierarchical Structure** - Organized by domain (config, core, plugins, features, ui)
* **Separated Concerns** - Plugin specs vs configs
* **Native ftplugin** - Filetype settings via Neovim's native mechanism
* **Embedded Antora Docs** - Documentation as code

== Quick Start

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-nvim ~/.config/nvim
nvim
----

Or run alongside existing config:

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-nvim ~/.config/nvim-domus
NVIM_APPNAME=nvim-domus nvim

# Create an alias
alias v='NVIM_APPNAME=nvim-domus nvim'
----

== Structure

[source]
----
domus-nvim/
├── init.lua                    # Entry point
├── lua/domus/                  # Configuration layers
│   ├── config/                 # User preferences
│   │   ├── options.lua
│   │   ├── keymaps.lua
│   │   └── autocmds.lua
│   ├── core/                   # Infrastructure
│   │   ├── lazy.lua
│   │   └── util.lua
│   ├── plugins/                # Plugin system
│   │   ├── specs/              # What to load
│   │   │   ├── editor.lua
│   │   │   ├── ui.lua
│   │   │   ├── coding.lua
│   │   │   ├── git.lua
│   │   │   ├── tools.lua
│   │   │   ├── debug.lua
│   │   │   └── lang/
│   │   └── config/             # How to configure
│   │       ├── treesitter.lua
│   │       ├── telescope.lua
│   │       ├── lsp/
│   │       ├── completion/
│   │       └── dap/
│   ├── features/               # Custom systems
│   │   └── motd/
│   └── ui/                     # Visual components
├── ftplugin/                   # Filetype settings
├── after/                      # Syntax overrides
└── docs/                       # Antora documentation
----

== Key Bindings

Leader: `<Space>`

=== Navigation

* `<Space>ff` - Find files
* `<Space>fg` - Live grep
* `<Space>e` - File browser (Oil)
* `<C-e>` - Harpoon menu

=== LSP

* `gd` - Go to definition
* `gr` - References
* `K` - Hover docs
* `<Space>ca` - Code action

=== Git

* `<Space>gs` - Git status
* `<Space>lg` - LazyGit
* `<Space>hs` - Stage hunk
