= domus-nvim
:description: A modular, layered Neovim 0.11+ configuration — 80 plugins, lazy.nvim, Catppuccin Mocha, full AsciiDoc authoring support
:icons: font
:url-docs: https://docs.architectus.dev
:url-repo: https://github.com/EvanusModestus/domus-nvim

A modular, layered Neovim configuration for Neovim 0.11+ — 80 plugins managed by lazy.nvim, built for
polyglot development: Lua, Python, Rust, C/C++, TypeScript, PowerShell, shell, SQL,
infrastructure-as-code, and technical writing in AsciiDoc and Markdown.

== Layout

The config is a single Lua tree; behaviour loads in order from `lua/domus/init.lua`:

[source,text]
----
lua/domus/
├── config/     -- options, keymaps, autocmds (loaded first)
├── core/       -- lazy.nvim bootstrap, highlights, utilities
├── plugins/
│   ├── specs/  -- what to load (by layer: editor, ui, tools, coding, git, debug, lang)
│   └── config/ -- how to configure each plugin
└── snippets/   -- LuaSnip tables
ftplugin/       -- asciidoc, c, lua, markdown, python
----

Plugin specs (what to load) live apart from plugin configs (how to configure) — adding a plugin never
touches configuration logic.

== Highlights

* *Completion* — blink.cmp (Rust engine: LSP, snippets, path, buffer)
* *LSP* — Mason + lspconfig, 17 servers auto-installed (lua_ls, pyright, rust_analyzer, clangd, ts_ls, powershell_es, sqlls, terraformls, …)
* *Format / lint* — conform.nvim (format-on-save) + nvim-lint
* *Syntax* — nvim-treesitter (24 parsers, textobjects); grammars fetched via git
* *Finder / files* — telescope.nvim (+ fzf, file-browser, project) and oil.nvim
* *Git* — gitsigns, lazygit, diffview
* *Debug* — nvim-dap with UI and virtual text
* *AI* — Copilot + CopilotChat
* *Theme* — Catppuccin Mocha with custom highlights (5 alternatives installed)

== Install

Choose *one* of the two setups below and stick with it.

[WARNING]
====
Do not mix them. Launching the same config sometimes as plain `nvim` and sometimes with
`NVIM_APPNAME` creates two separate data directories (`~/.local/share/nvim` vs
`~/.local/share/domus-nvim`). Mason and Treesitter bake absolute paths into their installs, so a
split leaves tool wrappers pointing at the wrong tree — LSP servers fail with exit code 127,
parsers vanish. Pick a lane; if you pick Option B, always launch through the alias.
====

=== Option A — dedicated (recommended)

domus-nvim becomes your Neovim: config at `~/.config/nvim`, data at `~/.local/share/nvim`, launched
with plain `nvim`. Simplest and least error-prone.

[source,bash]
----
# Back up any existing config + data first
mv ~/.config/nvim ~/.config/nvim.bak 2>/dev/null || true
mv ~/.local/share/nvim ~/.local/share/nvim.bak 2>/dev/null || true

git clone https://github.com/EvanusModestus/domus-nvim.git ~/.config/nvim
nvim
----

Prefer to keep the repo in a dotfiles/projects directory (and symlink it into place)? Same result,
still launched with plain `nvim`:

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-nvim.git ~/repos/domus-nvim
mv ~/.config/nvim ~/.config/nvim.bak 2>/dev/null || true   # ~/.config/nvim must not already exist
ln -s ~/repos/domus-nvim ~/.config/nvim
nvim
----

=== Option B — isolated (try it alongside your current config)

Keeps your existing `~/.config/nvim` untouched. `NVIM_APPNAME` namespaces both config and data.
Alias it so you *always* launch the same way — never run it plain, or you split its data dir.

[source,bash]
----
git clone https://github.com/EvanusModestus/domus-nvim.git ~/.config/domus-nvim
echo "alias vd='NVIM_APPNAME=domus-nvim nvim'" >> ~/.bashrc   # or ~/.zshrc
# from now on, ALWAYS launch with:  vd
----

=== First launch (either option)

Bootstraps lazy.nvim, downloads ~80 plugins, Mason installs servers/formatters, and Treesitter
compiles parsers (fetched via git). Let it finish before editing. Then verify inside Neovim:
`:Lazy` (plugins), `:Mason` (servers), `:checkhealth` (errors).

== Uninstall

Remove the paths for the option you installed.

[source,bash]
----
# Option A (dedicated). If ~/.config/nvim is a symlink, this removes the link, not the repo clone.
rm -rf ~/.config/nvim ~/.local/share/nvim ~/.local/state/nvim ~/.cache/nvim

# Option B (isolated)
rm -rf ~/.config/domus-nvim ~/.local/share/domus-nvim ~/.local/state/domus-nvim ~/.cache/domus-nvim
----
